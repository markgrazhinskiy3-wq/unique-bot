Important:
The visual appearance of the output media should remain almost the same as the original. Changes should be minimal and not noticeable to a typical user, but the file and encoding structure should differ due to processing and re-encoding steps.

Do NOT rewrite everything from scratch unless necessary. Improve and extend the existing codebase.

PART 4 — RANDOM PIPELINE GENERATOR
================================

Implement a random pipeline generator:

For each media file:
1. Load preset
2. Randomly decide which operations to apply based on probabilities
3. Randomly generate parameters within ranges
4. Build processing pipeline
5. Execute pipeline
6. Validate output

The pipeline should not always apply the same filters.
Each run should produce slightly different processing parameters.
