Create a Telegram bot in Python that uniqualizes photos and videos for Facebook ads without visible changes for the human eye. The goal is to make Facebook's algorithms see each processed file as a completely new creative.

Tech Stack: Python 3.11+, python-telegram-bot (version 20+, async), Pillow (PIL), OpenCV (cv2), numpy, FFmpeg (system package) + ffmpeg-python, piexif.

Bot Flow: User sends /start, bot replies "👋 Привет! Я бот для уникализации креативов. Отправь мне фото или видео, и я сделаю его уникальным для Facebook. 📸 Фото — отправляй как файл (документ) для лучшего качества. 🎬 Видео — отправляй как файл (документ). Можно отправлять и обычным способом (со сжатием)." User sends a photo or video as media or document. Bot replies "⏳ Уникализирую файл, подождите..." Bot processes the file applying ALL uniqualization techniques. Bot sends back the processed file as document to preserve quality. Bot replies "✅ Готово! Файл уникализирован. Facebook увидит это как новый креатив."

Photo Uniqualization — apply ALL in this order. Step 1 Metadata destruction: strip ALL EXIF, XMP, IPTC data completely, remove embedded thumbnails, remove ICC color profiles and re-embed generic sRGB, remove ALL comments and markers. Step 2 Pixel grid shift: add 2px border on all sides using edge pixel colors reflect mode, crop 1-3 pixels from each side random per side not symmetric, scale back to dimensions plus or minus 1-2px from original not exact original size. Step 3 Sub-pixel rotation: rotate by random -0.3 to +0.3 degrees, auto-crop black borders, this redistributes pixel values through interpolation and breaks pHash. Step 4 Color manipulation: shift hue by random -1.0 to +1.0 degrees, shift brightness by random -2 to +2 out of 255, shift contrast by random factor 0.98-1.02, shift saturation by random factor 0.98-1.02, per-channel R G B independent random brightness shift of -1 to +1. Step 5 Noise injection: Gaussian noise sigma 1-2 out of 255, subtle multiplicative film grain 0.5-1%, non-uniform more noise in dark areas less in bright. Step 6 Invisible watermark: random UUID string rendered at random position, opacity 0.3-0.5% and font size 8-14px. Step 7 Micro geometric transforms: zoom in by random 0.5-1.5% crop from center, subtle perspective warp with corners shift 0.5-1px randomly. Step 8 Frequency domain manipulation: minimal Gaussian blur kernel 0.1-0.3px, then subtle unsharp mask to compensate, net visual effect zero but frequency spectrum different. Step 9 Pixel-level uniqueness: 20-50 random 2x2 blocks shift R G values by plus or minus 1, first and last row of pixels shift each value by random -1 to +1. Step 10 Color gradient overlay: subtle linear gradient with random colors and random direction, applied at 0.3-0.5% opacity. Step 11 Re-encoding: JPEG random quality 93-97, PNG random compression level, no metadata written during save, filename unique_{uuid4}.{ext}. Step 12 Final verification: verify zero metadata in output, log MD5 hash difference from input.

Video Uniqualization — apply ALL. Step 1 Probe original: ffprobe to get resolution fps bitrate codec audio codec duration. Step 2 Metadata destruction: FFmpeg -map_metadata -1, remove chapters subtitles attachments thumbnails. Step 3 Visual filters as FFmpeg filter chain: crop 2-4px from random edges not symmetric, scale back to resolution plus or minus 2px from original, noise c0s=2 c0f=t, brightness plus or minus 0.5% eq=brightness=0.003, contrast plus or minus 0.5% eq=contrast=1.005, saturation plus or minus 0.5% eq=saturation=1.005, hue plus or minus 0.5 degrees hue=h=0.5, subtle unsharp=3:3:0.3:3:3:0.1. Step 4 Temporal manipulation: FPS micro-shift 30 to 29.97 or 30.02. Step 5 Encoding manipulation: H.264 libx264, CRF plus or minus random 1-2 from equivalent quality, random GOP size 48 or 72, different B-frames count 2 or 3. Step 6 Audio manipulation: re-encode AAC at slightly different bitrate plus or minus 8-16kbps, shift audio by 1-5ms, if no audio add silent track. Step 7 Speed micro-adjustment: plus or minus 0.1-0.3% speed change setpts=PTS*1.002. Step 8 Output: filename unique_{uuid4}.mp4, verify zero metadata, clean up temp files.

Technical Requirements: all changes invisible to human eye. File hash MD5 SHA must be completely different from original. Handle errors gracefully with try except. Use /tmp or tempfile for processing and clean up after sending. Bot token from environment variable TELEGRAM_BOT_TOKEN. Support photos sent as compressed photo or document jpg png webp. Support videos sent as compressed video or document mp4 mov avi. Accept any file size without restrictions and process whatever the user sends. No file size validation or rejection. Logging for each processing step. Async processing and run heavy processing in thread pool executor. Polling mode not webhooks.

File Structure: main.py, processors/__init__.py, processors/image_processor.py, processors/video_processor.py, utils/__init__.py, utils/metadata.py, utils/helpers.py, requirements.txt, Dockerfile, README.md.

Dockerfile needed for FFmpeg: FROM python:3.11-slim, RUN apt-get update and apt-get install -y ffmpeg libgl1-mesa-glx libglib2.0-0, WORKDIR /app, COPY requirements.txt and RUN pip install --no-cache-dir -r requirements.txt, COPY . ., CMD python main.py.

requirements.txt: python-telegram-bot==20.7, Pillow==10.2.0, numpy==1.26.3, opencv-python-headless==4.9.0.80, ffmpeg-python==0.2.0, piexif==1.1.3.

Environment Variables: TELEGRAM_BOT_TOKEN set in Replit Secrets.

Error messages in Russian: wrong format "⚠️ Неподдерживаемый формат. Отправьте фото или видео.", processing error "❌ Ошибка обработки. Попробуйте ещё раз.", timeout "⏱ Слишком долгая обработка. Попробуйте файл поменьше."

Make sure FFmpeg is available via Dockerfile or Replit nix packages pkgs.ffmpeg in replit.nix.
